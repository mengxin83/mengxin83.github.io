Spark on Yarn部署

发表于 2020-02-09 | 分类于 大数据 |

Spark部署模式

Spark共有五种部署模式:Standalone、YARN、Kubernetes、Amazon EC2、Mesos，比较常用的是YARN和Kubernetes，这里介绍Spark on YARN部署。本文档介绍HA版本的Hadoop 3.2.1和Spark 2.4.4的整合部署。

准备

  • 硬件

分布式部署至少需要2台服务器，否则通过Spark提交任务到YARN时会报找不到ResourceManager的错误。

  • 需要的软件及版本
软件 版本 备注
Centos 7.6.1810 64位版本
jdk-8u241-linux-x64.tar.gz 8u241 最好Java 8以上版本
scala-2.11.12.tgz 2.11.12 官网上Spark二进制安装包对应Scala版本不同，要看清楚
apache-zookeeper-3.5.6-bin.tar.gz 3.5.6
hadoop-3.2.1.tar.gz 3.2.1
spark-2.4.4-bin-hadoop2.7.tgz 2.4.4 可以自己编译也可以使用官方编译好的版本，有集成Hadoop和没有集成Hadoop的安装包；该版本也兼容Hadoop 3
python 3.6.8 运行pyspark需要
hive 3.1.2 Spark Beeline、spark-sql依赖hive metastore
mysql
  • 角色分配

主机1:Zookeeper、NameNode、DataNode、ZKFC、JournalNode、JobHistoryServer、ResourceManager、NodeManager、hive metastore

主机2:NameNode、DataNode、ZKFC、ResourceManager、NodeManager

通过Spark on YARN模式提交任务不需要启动Spark的Master和Worker服务

  • 安装路径

所有服务器安装路径，用户相同

安装用户:deep 所有系统密码统一，不要使用root部署和启动服务

安装根路径:/home/deep/bigdata (自己规划，可放在/opt下)
